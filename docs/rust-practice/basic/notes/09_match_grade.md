# 09 Match Grade - 补充笔记

## 本题覆盖

- `match` 表达式。
- 精确值和范围模式。
- `_` 通配符。
- 匹配 `Option<T>`。
- 返回 `&'static str`。

## 需要补充

### `match` 也是表达式

`match` 可以直接返回值：

```rust
fn grade(score: i32) -> &'static str {
    match score {
        90..=100 => "A",
        _ => "invalid",
    }
}
```

每个分支的返回类型必须一致。

### 分支顺序有意义

`match` 从上到下检查。范围不要互相覆盖，否则后面的分支可能永远匹配不到。

本题的分数区间互不重叠：

- `90..=100`
- `80..=89`
- ...
- `0..=59`

### `_` 表示“其他所有情况”

`_` 通常放在最后，兜底处理未列出的值。

对于分数，负数和大于 100 的值都应该进入 `_`，返回 `"invalid"`。

### 字符串字面量是 `&'static str`

`"A"`、`"invalid"` 这类字符串字面量存放在程序二进制中，生命周期是 `'static`。

所以函数可以返回：

```rust
fn grade(score: i32) -> &'static str
```

这里没有分配新的 `String`。

## 常见坑

- 忘记处理无效分数。
- `_` 放得太早，导致后续分支不可达。
- `Some(value)` 分支里忘记调用 `grade(value)`。
