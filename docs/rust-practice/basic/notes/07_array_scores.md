# 07 Array Scores - 补充笔记

## 本题覆盖

- 固定长度数组 `[T; N]`。
- 下标访问。
- `.len()`。
- `for` 循环遍历数组。
- 简单求和。

## 需要补充

### 数组长度是类型的一部分

`[i32; 5]` 和 `[i32; 4]` 是不同类型。

```rust
fn print_scores(scores: [i32; 5])
```

这个函数只接收长度为 5 的 `i32` 数组。

### 数组和 `Vec` 的区别

- 数组长度固定，编译期已知。
- `Vec<T>` 长度可变，数据在堆上增长。

本题使用数组，是为了练习固定长度集合，不涉及动态增删。

### 下标访问可能 panic

```rust
scores[0]
scores[4]
```

只要下标在范围内就可以。越界访问会在运行时 panic。

更安全的访问方式是 `.get(index)`，返回 `Option<&T>`，但本题先练习直接下标。

### `[i32; 5]` 可以复制

`i32` 是 `Copy`，固定长度数组在元素都是 `Copy` 时也可以 `Copy`。

所以：

```rust
print_scores(scores);
sum_scores(scores);
```

不会造成所有权问题。若数组元素是 `String`，情况就不同。

## 常见坑

- 以为 `[i32; 5]` 和 `[i32; 4]` 都是同一种数组。
- 下标从 1 开始算，导致访问最后一个元素写错。
- 忘记直接下标越界会 panic。
