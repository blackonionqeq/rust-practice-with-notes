# 67 Macro vs Function - 笔记

## 本题会用到的前置知识点

- 函数签名设计。
- 字符串格式化。
- 宏的基本语法。

## 核心结论

- 能用函数清楚表达时，优先函数。
- 宏适合语法级复用，不是默认抽象工具。
- 选择标准：可读性、可调试性、维护成本。

## 核心示例

```rust
fn to_kv_line(k: &str, v: &str) -> String {
    format!("{}={}", k, v)
}

fn main() {
    println!("{}", to_kv_line("mode", "dev"));
}
```

理解重点：这个场景函数足够清晰，宏收益很小。

## 常见坑

- 为了“看起来高级”而滥用宏。
- 不比较函数方案就直接上宏。

## 回看问题

- 这个问题必须要语法级能力吗？
- 换成函数后，是否更容易测试和维护？
