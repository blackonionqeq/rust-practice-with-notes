# 63 macro_rules Basic - 笔记

## 本题会用到的前置知识点

- 函数与参数。
- `println!` 格式化输出。
- 表达式基础。

## 核心结论

- `macro_rules!` 可做“语法层”复用。
- 入门阶段先掌握：匹配一个表达式参数并展开。
- 宏不等于函数；它先展开，再参与编译。
- 宏调用通常带 `!`，例如 `say_hello!("rust")`。
- `$name:expr` 中的 `expr` 是 matcher fragment，表示匹配一个 Rust 表达式。

## 需要补充看懂的知识

### 宏是“展开成代码”

函数调用是在已有函数签名下传值；宏调用更早发生：编译器先把宏调用展开成 Rust 代码，再继续类型检查和编译。

```rust
say_hello!("rust");
```

可以粗略理解为先展开成：

```rust
println!("hello, {}", "rust");
```

然后这段展开后的代码再参与正常编译。入门阶段不需要会看真实展开结果，但要知道：宏不是运行时函数调用。

### `macro_rules!` 的基本形状

```rust
macro_rules! say_hello {
	($name:expr) => {
		println!("hello, {}", $name);
	};
}
```

可以分成三块看：

- `say_hello`：宏名。
- `($name:expr)`：匹配调用方传进来的语法。
- `{ println!(...) }`：匹配成功后展开出的代码。

`$name` 不是普通变量，而是宏匹配到的一段语法。`:expr` 表示它必须能作为表达式出现。

### 为什么调用要写 `!`

```rust
say_hello!("rust");
```

`!` 明确告诉读者和编译器：这里调用的是宏，不是函数。宏可以接收更灵活的语法片段，甚至像 `println!` 那样接收可变数量参数；函数则必须走固定的参数列表和类型检查。

### 宏体里要注意分号和表达式

本题的宏只是打印，所以展开成语句即可：

```rust
println!("hello, {}", $name);
```

如果宏要返回一个值，通常会用双层花括号形成一个块表达式：

```rust
macro_rules! make_text {
	($value:expr) => {{
		let out = format!("{}", $value);
		out
	}};
}
```

第 63 题不用写这种返回值宏，但先知道这种写法能避免后面看到 `{{ ... }}` 时困惑。

## 核心示例

```rust
macro_rules! say_hello {
    ($name:expr) => {
        println!("hello, {}", $name);
    };
}

fn main() {
    say_hello!("rust");
}
```

理解重点：`$name:expr` 表示“匹配一个表达式”。

## 常见坑

- 把宏当函数看，忽略“先展开”的过程。
- 忘记调用宏时要写 `!`。
- 不理解 `$name` 是匹配到的语法片段，不是普通函数参数。
- 宏体写太复杂，反而难读。
