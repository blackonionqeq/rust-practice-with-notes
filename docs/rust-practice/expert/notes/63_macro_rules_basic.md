# 63 macro_rules Basic - 笔记

## 本题会用到的前置知识点

- 函数与参数。
- `println!` 格式化输出。
- 表达式基础。

## 核心结论

- `macro_rules!` 可做“语法层”复用。
- 入门阶段先掌握：匹配一个表达式参数并展开。
- 宏不等于函数；它先展开，再参与编译。

## 核心示例

```rust
macro_rules! say_hello {
    ($name:expr) => {
        println!("hello, {}", $name);
    };
}

fn main() {
    say_hello!("rust");
}
```

理解重点：`$name:expr` 表示“匹配一个表达式”。

## 常见坑

- 把宏当函数看，忽略“先展开”的过程。
- 宏体写太复杂，反而难读。

## 回看问题

- 这个场景必须用宏吗？
- 宏参数匹配规则是否一眼能看懂？
